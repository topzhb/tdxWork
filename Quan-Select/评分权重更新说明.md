## 修改原因

### 1. 综合评分权重调整
- **加强基本面重要性**：原60%:40%的比例技术面权重过高，调整为50%:50%更平衡
- **提高筛选质量**：基本面评分<50分的股票通常存在财务问题或成长性不足，直接过滤可提高精选池质量
- **符合价值投资理念**：基本面在长期投资中更为重要，增加其权重符合价值投资原则

### 2. 技术面评分结构调整
- **加强趋势重要性**：趋势结构在技术分析中最为关键，反映了股票的整体方向
- **降低位置形态权重**：位置相对次要，且容易被短期波动影响
- **保持动量重要性**：动量指标反映短期动能，对交易时机有指导意义
- **保持量能重要性**：量能确认价格趋势的有效性

## 预期效果

### 1. 综合效果
- **精选结果更优质**：剔除基本面过弱的股票
- **评分更平衡**：技术面和基本面同等重要
- **风险控制更好**：避免因技术面好看而买入基本面差的股票

### 2. 技术面分析效果
- **趋势权重更大**：趋势良好的股票评分更高
- **更重视方向**：减少对位置形态的依赖，更关注趋势方向
- **更符合技术分析原理**：趋势为王，位置次之

## 回测模块说明

### 回测与精选模块的区别
1. **评分体系一致**：使用相同的技术面评分结构和综合评分权重
2. **过滤规则不同**：
   - **精选模块**：应用基本面<50分过滤，剔除基本面过弱股票
   - **回测模块**：不应用基本面过滤，客观呈现历史评分
3. **目的不同**：
   - **精选模块**：生成当前最优投资建议
   - **回测模块**：验证历史评分的有效性，保持客观性

### 回测模块更新内容
1. **技术面评分结构**：更新为趋势40+位置15+动量25+量能20
2. **综合评分权重**：更新为技术×0.5 + 基本面×0.5
3. **显示详细理由**：包括基本面低分的具体原因说明

### 2. Surprise-only 模式逻辑优化

**核心变更：**
- `--surprise-only` 参数现在**强制使用纯 surprise 策略**（非组合）
- 纯 surprise 策略时**不限TOP数量**，输出全部 diff>0 的命中股票
- 组合策略含 surprise（如 `classic,surprise`）时**限制TOP30**

**使用方式：**
```bash
# 纯 surprise 策略，不限TOP，仅输出超预期股票
python main.py --surprise-only

# 组合策略，限制TOP30
python main.py --fund-strategy classic,surprise

# 纯基本面 + surprise，限制TOP30
python main.py --skip-tech --fund-strategy surprise
```

**代码修改位置：**
1. `main.py`：判断 `is_pure_surprise = (args.fund_strategy == "surprise")`
2. `score.py`：`surprise-only` 时强制 `strat = "surprise"`
3. `report.py`：`surprise_only=True` 时不截断TOP

## 使用建议
